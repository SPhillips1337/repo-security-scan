# Configuration Scan Results

## Project Identification
- **Project Type:** Python CLI Application (Security Scanner)
- **Language:** Python 3
- **Purpose:** Scans local repository directories for hardcoded secrets and sensitive data with configurable patterns and alerting.

## Available Commands / Scripts

### 1. Python CLI (`main.py`)
```bash
python main.py [OPTIONS] [TARGET_DIR]
```

| Option              | Description                                       | Default     |
|---------------------|--------------------------------------------------|-------------|
| `target`            | Directory to scan                                | `.`         |
| `--config`, `-c`    | Path to a YAML configuration file                | None        |
| `--verbose`, `-v`   | Enable verbose output                            | False       |
| `--scan-mode`       | Scan mode: `full` or `quick`                     | `full`      |

**Exit Codes:**
- `0` — No critical findings
- `1` — Critical secrets detected
- `2` — Error

### 2. Shell Script (`scan.sh`)
```bash
./scan.sh <repo-url> [--deep] [--output-dir <dir>]
```

| Parameter     | Description                                    | Default           |
|--------------|------------------------------------------------|-------------------|
| `repo-url`   | Git repository URL to clone and scan            | (required)        |
| `--deep`     | Deep scan all text files vs. quick targeted scan | `false`         |
| `--output-dir` | Directory for scan reports                   | `./scan-reports`  |

## Configuration Structure

### Schema: `config/schema.yaml`
Defines the YAML configuration file format with top-level keys:
- **targets** — Repository paths to scan
- **exclusions** — Directories, extensions, size limits to ignore
- **patterns** — Built-in pattern categories + custom regex rules
- **severity** — Minimum severity threshold (`CRITICAL`, `HIGH`, `MEDIUM`, `LOW`)
- **output** — Format (`text`/`json`/`csv`/`sarif`), file output, alerting prefs
- **validation** — Pre-scan validation rules

### Supported Secret Pattern Categories
`aws`, `github`, `generic_api_key`, `password`, `private_key`, `cloud`
